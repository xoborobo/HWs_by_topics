Структура баг-репорта:

1. Короткое описание (Summary)
2. Проект (Project)
3. Компонент приложения (Component)
4. Номер версии (Version)
5. Серьезность (Severity)
6. Приоритет (Priority)
7. Статус (Status)
8. Автор (Author)
9. Назначен на (Assigned To)
10. ОС / Сервис Пак и т.д. / Браузера + версия / ...
11. Шаги воспроизведения (Steps to Reproduce)
12. Фактический Результат (Result)
13. Ожидаемый результат (Expected Result)
14. Прикрепленный файл (Attachment)
